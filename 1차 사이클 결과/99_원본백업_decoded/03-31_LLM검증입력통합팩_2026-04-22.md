# 2026-04-22 LLM 검증 입력 통합팩

## 문서 목적

이 문서는 외부 `LLM(대규모 언어모델)`이 현재 프로젝트를 한 번에 검증할 수 있도록, `문제상황(problem situation, 원문 과제 정의)`, `로드맵(roadmap, 전체 진행표)`, `판단기준(criteria, 판단 축과 게이트)`, `현재상태(current state, 지금까지의 판정)`, `향후계획(next plan, 다음 실행 계획)`을 하나로 묶은 입력 문서다.

이 문서는 `최종 작물 추천` 문서가 아니다.  
이 문서의 목적은 현재 판단과 다음 단계가 **방법론적으로 맞는지** 검증받는 것이다.

## 1. 문제상황

프로젝트의 원문 문제상황은 다음과 같다.

- 주체: 국내 종자회사 `N사`
- 역할: 작물 유전체 연구실 연구원
- 과제: 최근 국내외에서 수요가 증가하는 작물의 `표준유전체(reference genome, 기준 유전체)` 연구 프로젝트를 `5주` 안에 기획
- 산출물: 연구수행 전략 기획안 + `PPT` 30분 발표
- 전제: 실제 실험 수행이 아니라 `기획(project planning, 전략 설계)`이 목적
- 범위: 국내 회사이지만 타겟 시장을 한국으로 제한하지 않음

핵심 해석:

1. 단순히 유명한 작물을 뽑는 과제가 아니다.
2. `수요 증가 작물`이어야 한다.
3. 그 수요가 `새 기준 유전체 프로젝트` 기획으로 이어져야 한다.
4. `5주 기획 과제`로 감당 가능해야 한다.

## 2. 프로젝트 로드맵

프로젝트 로드맵은 크게 두 `cycle(사이클, 큰 작업 묶음)`로 구성된다.

### Cycle 1. 작물 선정

목표:

- 최종 작물 1개 확정
- 선정 근거 `brief(브리프, 짧은 설명 문서)` 확보

현재 작업의 위치:

- 아직 `scenario close(시나리오 잠금)` 전이다.
- 아직 `crop-group long-list(작물군 긴 후보 목록)` 단계로 넘어가지 않았다.

### Cycle 2. 유전체 해독 설계

Cycle 1이 끝난 뒤에만 들어간다.

- 유전체 특성 `deep-dive(심층 조사)`
- 시퀀싱/조립 전략 설계
- 발표 통합

핵심 운영 해석:

1. Cycle 1이 끝나야 Cycle 2로 간다.
2. 지금은 아직 작물 선정을 완전히 끝낸 상태가 아니다.
3. 지금은 `시나리오 검증과 잠금`이 우선이다.

## 3. 판단기준

현재 판단기준은 `hard gate(강한 통과 조건)`와 `축(axis, 비교 축)`을 분리해서 읽는다.

### 3-1. 최상위 원칙

1. `PBL 원문 적합성`이 최우선이다.
2. `이미 공개 유전체가 충분한 작물`은 우선 탈락이다.
3. `reference-genome necessity(새 기준 유전체 필요성)`는 soft score가 아니라 hard gate다.
4. `beneficiary(수혜자)`, `buyer(구매자)`, `payer(예산 주체)`, `decision-maker(의사결정자)`를 분리한다.
5. `public demand exists(공공 수요가 있음)`를 `exact buyer lock exists(정확한 구매자 고정)`로 과잉 해석하지 않는다.

### 3-2. 시나리오 Hard Gate

시나리오 수준에서 보는 `SG-1 ~ SG-8` 기준이 있다.

특히 중요한 축:

- `SG-6`: 법·제도 경로
- `SG-7`: genome 필요성

규칙:

- `N` 하나라도 있으면 시나리오는 폐기
- `B`, `UNKNOWN`, `CONFLICT`면 보완 전까지 long list 생성 금지

### 3-3. 작물 × 시나리오 Hard Filter

작물 단위로는 `HF-0 ~ HF-9`를 본다.

핵심:

- `HF-0`: 공개 유전체 충분성 fail-fast
- `HF-3`: 기존 자원 대비 novelty
- `HF-4`: genome-to-breeding bridge
- `HF-7`: genome technical feasibility

중요 규칙:

- `B`, `UNKNOWN`, `CONFLICT`는 soft score로 보내지 않는다.
- `HF-0`, `HF-3`, `HF-4`, `HF-7` 중 핵심이 안 닫히면 점수화로 보내면 안 된다.

### 3-4. 현재 프로젝트의 운영 축

현재 실무 운영에서는 아래 `G1 ~ G8` 구조를 쓴다.

- `G1`: end-use fit
- `G2`: buyer / payer specificity
- `G3`: regional heterogeneity
- `G4`: trade / import / price regime
- `G5`: seed channel / replacement / value capture
- `G6`: regulation / sovereignty / ABS / governance
- `G7`: policy durability
- `G8`: operational counterparty / transaction object

현재 프로젝트의 중요한 추가 규칙:

1. `HF-0 / HF-3 / HF-4 / HF-7`는 `pre-BWM hard gate(점수화 전 선행 게이트)`다.
2. `G2`와 `G8`은 반드시 분리한다.
3. `BWM(최선-최악 비교 가중치 방식)`은 hard gate 통과 후에만 의미가 있다.

## 4. 현재 상태

현재 live scope는 아래 3개 lane만 유지한다.

- `N1 x Chloris gayana`
- `N2 x Lens culinaris`
- `N2 x Vigna mungo`

중요:

- 예전 `legacy ranking(과거 순위 메모)`은 참고일 뿐, 현재 live run의 최종 순위가 아니다.
- `P1`, reserve lane, global reranking은 현재 live scope 밖이다.

### 4-1. 공통 판정

`R2` 기준 공통 판정은 이렇다.

1. 세 lane 모두 살아 있다.
2. 세 lane 모두 아직 `pending-gate`다.
3. 세 lane 모두 아직 `score-ready`는 아니다.
4. 따라서 아직 `scenario close`를 못 한다.
5. 따라서 아직 `final crop selection`도 못 한다.

### 4-2. N1 x Chloris gayana

좋아진 점:

- `public genome exists or not`가 아니라, `Kenya cultivar-matched breeding-decision benchmark`에서 현재 공개 자원이 충분한가로 질문이 좁혀졌다.
- legal route는 `NEMA-only`보다 `NEMA + NACOSTI`가 더 강한 provisional baseline으로 올라왔다.

여전히 막는 점:

- `HF-0 / HF-3 / HF-4`가 아직 `pass`가 아니다.
- `G6` direct official conflict가 남아 있다.
- exact source-holder, carve-out, owner chain이 안 닫혔다.
- `G2 / G8 / G5`가 아직 비어 있다.

### 4-3. N2 x Lens culinaris

좋아진 점:

- lentil-specific named buyer/program line이 있다는 점까지는 올라왔다.
- `G6`은 `pending-gate but operationally favorable` 수준으로 개선됐다.

여전히 막는 점:

- exact sanctioned budget holder가 안 닫혔다.
- `technical user`가 분리 잠금되지 않았다.
- buyer-side route는 정책 의존성이 강하다.
- `HF-0 / HF-3 / HF-4 / G5 / G6`는 아직 `pass`가 아니다.

### 4-4. N2 x Vigna mungo

좋아진 점:

- current official procurement / supply-management continuity가 더 분명해졌다.
- provenance structure와 일부 holder-side official route가 개선됐다.

여전히 막는 점:

- downstream end-buyer closure가 부족하다.
- `technical user`, `acceptance criterion`이 안 닫혔다.
- India-side national route closure가 부족하다.
- `HF-0 / HF-3 / HF-4 / G5 / G6`는 아직 `pass`가 아니다.

## 5. 향후 계획

지금 필요한 건 넓은 재탐색이 아니라, lane별 hard gate를 실제로 닫는 것이다.

### 5-1. 바로 다음 단계

#### N1

- benchmark-specific insufficiency chain 닫기
- `G6` official conflict 닫기
- source-holder / carve-out / owner chain 닫기

#### N2-Lens

- exact budget holder 닫기
- technical user 닫기
- exact working-route closure 확보

#### N2-Urd

- downstream buyer lock 닫기
- technical user / acceptance criterion 닫기
- India-side national route closure 확보

### 5-2. 그 다음 단계

위 reopen이 끝나면:

1. `gate ledger(게이트 표)`를 다시 갱신
2. 어떤 lane이 실제로 `pass`에 가까워졌는지 확인
3. 그 다음에만 `scenario close` 가능 여부 판정

### 5-3. scenario close 이후 단계

scenario close가 가능해지면 그 다음 순서는 이렇다.

1. `crop-group long-list`
2. `crop x scenario hard filter`
3. `deep crop analysis(심층 작물 분석)`
4. `final crop selection`

## 6. 현재 검증하고 싶은 핵심 질문

외부 `LLM`이 이번 문서를 읽고 특히 검증해줬으면 하는 질문은 아래다.

1. 현재 프로젝트가 아직 `scoring`으로 가면 안 된다는 판정이 방법론적으로 맞는가
2. `HF-0 / HF-3 / HF-4 / HF-7`를 `pre-BWM hard gate`로 둔 현재 구조가 맞는가
3. `G2`와 `G8`을 분리해야 한다는 현재 판단이 충분히 정당한가
4. 현재 live scope를 `N1`, `N2-Lens`, `N2-Urd`로 유지하는 것이 맞는가
5. 지금 단계에서 가장 합리적인 다음 행동이 `targeted reopen(정밀 재오픈)`인지, 아니면 다른 루트인지
6. 현재 남은 blocker 중 어떤 것이 진짜 결정적이고, 어떤 것은 부차적인가

## 7. 검증 시 금지했으면 하는 것

외부 `LLM`은 아래 anti-pattern(안 좋은 패턴)을 피해야 한다.

1. `public demand exists`를 `exact buyer lock exists`로 올려 읽지 말 것
2. `existing public genome exists`를 자동으로 `genome necessity fail` 또는 `auto-pass`로 읽지 말 것
3. hard gate가 안 닫힌 lane을 soft score 비교로 밀어 넣지 말 것
4. `N1 > N2` 또는 `Lens > Urd` 같은 결론을 evidence 부족 상태에서 먼저 닫지 말 것
5. `final crop recommendation`으로 점프하지 말 것

## 9. 한 줄 요약

현재 프로젝트는 `후보를 많이 줄인 상태`가 아니라, `남은 3개 lane은 살아 있지만 아직 hard gate가 안 닫혀서 점수화와 최종 선정으로 넘어가면 안 되는 상태`다. 지금 필요한 건 넓은 재탐색이 아니라, lane별 hard gate를 실제로 pass로 바꿀 마지막 증거를 정밀하게 채우는 것이다.
