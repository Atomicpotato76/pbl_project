# 프로젝트 폴더 안내

이 루트는 이제 `현재 기준본`, `시나리오 검증`, `운영지원`, `엔지니어링`, `파이프라인`으로 읽으면 된다.
형주가 헷갈리기 쉬웠던 `문서허브 / 기록 / 배포패키지`는 `운영지원_통합` 아래 3갈래로 합쳤다.

## 루트 폴더 역할

- `최신 상황`: 지금 바로 참고하는 활성 기준본 묶음
- `시나리오 검증용`: 시나리오 검증 실험 워크스페이스
- `운영지원_통합`: 안내 허브, 작업기록, 배포세트를 함께 모아 둔 지원 폴더
- `엔지니어링`: 프롬프트 설계와 사고 프레임 관련 가이드
- `Self_Working_pipeline`: 실행 코드와 산출물 파이프라인

## 바로 보기

- `운영지원_통합/INDEX.md`: `문서안내허브 / 작업기록 / 배포세트` 차이를 먼저 설명하는 통합 입구
- `최신 상황/INDEX.md`: 지금 기준으로 봐야 하는 문서 지도
- `시나리오 검증용/INDEX.md`: 검증용 입력, 프롬프트, 판정 흐름, N1 심화 문서 지도
- `운영지원_통합/01_문서안내허브/INDEX.md`: 과거 판단 과정 / 새 기준 문서 / 중복 기준본 허브
- `운영지원_통합/02_작업기록/INDEX.md`: 과거 자료와 세션 기록 안내
- `운영지원_통합/03_배포세트/INDEX.md`: 목적별 첨부 세트 설명
- `엔지니어링/INDEX.md`: 프롬프트 엔지니어링 가이드 안내

## 정리 원칙

1. 루트에는 가급적 폴더만 둔다.
2. 같은 문서가 여러 군데 필요하면 `기준본 1개 + 검증/배포/보관 복사본` 구조로 읽는다.
3. 현재 문서는 `최신 상황`, 실험/검증 문서는 `시나리오 검증용`, 과거 흐름과 배포 세트는 `운영지원_통합`에서 찾는다.
4. 외부 제출용 세트는 새로 루트에 흩뿌리지 말고 `운영지원_통합/03_배포세트` 아래에 추가한다.
5. 중복이 의심되면 먼저 `운영지원_통합/01_문서안내허브/중복문서_기준본_매핑.md`에서 기준본 위치를 확인한다.
