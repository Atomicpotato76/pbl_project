# 폴더 구조 설명

`최신 상황`은 지금 기준으로 살아 있는 문서만 남기는 작업 구역이다.  
헷갈리지 않게 폴더는 3개만 두고, 각 폴더 안에서는 `번호-번호_문서성격_주제` 형식으로 통일한다.

## 정리 원칙

- 루트에는 대분류 폴더와 안내 문서만 둔다.
- 이름에서 먼저 문서 성격이 보이게 한다.
  예: `기준하네스`, `사전조사`, `판정결과`, `통합현황`, `검증프롬프트`
- 과거 버전과 복사본은 `최신 상황`에 두지 않는다.
- 번호를 붙여 읽는 순서와 문서 위계를 바로 보이게 한다.

## 폴더 구조

- `01_프로젝트기반과활성기준`
  프로젝트 문제 정의, 운영 결정, 상위 로드맵, 활성 기준, 후속 로드맵

- `02_MC1-1_작물선정진행`
  MC1-1에서 실제로 쓰는 기준 하네스, 조사 문서, 판정 결과, 통합 현황, 쉬운 설명 문서

- `03_검증프롬프트와설정`
  검증 배경, 검증 프롬프트, 클로드용 템플릿, 생성 결과, 레드팀 프롬프트, 설정파일, 생성 스크립트
