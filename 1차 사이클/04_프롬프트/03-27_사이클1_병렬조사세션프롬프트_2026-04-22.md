# 2026-04-22 사이클 1 병렬 조사 세션 프롬프트

## 목적

이 문서는 `사이클 1(cycle 1, 첫 번째 작업 묶음)`의 병렬 조사 세션에 바로 붙여넣어 실행할 수 있는 `프롬프트(prompt, 세션 지시문)` 모음이다.

원칙은 세 가지다.

1. 각 세션은 자기 번들 범위만 다룬다.
2. `worker(워커, 개별 조사 세션)`는 자기 산출물만 쓰고, `coordinator(코디네이터, 총괄 세션)`만 통합 문서를 쓴다.
3. `working / provisional(작업용 / 임시)`과 `final(최종 확정)`을 절대 섞지 않는다.

---

## 실행 순서 요약

1. 먼저 `C0`를 실행한다.
2. 그다음 `P1~P5`를 동시에 실행한다.
3. 결과가 모이면 `I1`, `I2`, `I3`를 실행한다.
4. 마지막에 `R1`을 실행한다.

---

## 공통 규칙

아래 블록은 모든 세션 프롬프트 맨 위에 공통으로 붙는다.

```md
## 불변 앵커(invariant anchor, 전체 세션에서 유지할 고정 조건)
- 현재 단계: cycle 1 pre-BWM evidence collection
- live scope(현재 실제 범위): N1 x Chloris gayana, N2 x Lens culinaris, N2 x Vigna mungo only
- out-of-scope(범위 밖): fresh global reranking, Brazil/Morocco reserve lane, final crop recommendation
- 이번 세션 목표: 내 번들 범위 안의 evidence artifact(증거 산출물)만 완성
- methodology guardrail(방법론 안전장치):
  - HF-0 / HF-3 / HF-4 / HF-7는 pre-BWM hard gate로 다룬다
  - G2와 G8은 반드시 분리한다
  - FACT / INFERENCE / HYPOTHESIS / UNKNOWN을 섞지 않는다
  - public demand exists를 exact buyer lock exists로 과잉 해석하지 않는다
  - existing public resource exists 여부를 먼저 본 뒤 genome necessity를 논한다
- 산출물 상태 표기: provisional only, not final
```

## 이 문서로 어디까지 가는가

이 프롬프트 묶음은 `작물 선정(crop selection, 최종 작물 1개 확정)` 자체를 바로 끝내는 문서가 아니다.

이 문서로 끝낼 수 있는 범위는 여기까지다.

1. `pre-BWM evidence collection` 완료
2. `partial revalidation` 1회 완료
3. `N1/N2` 혹은 `N2 내부 carry candidate`의 상태 명확화
4. `scenario close(시나리오 잠금)` 직전 또는 직후 단계 진입

즉, 이 세션들을 다 돌리고 이 세션에서 종합하면 `시나리오 쪽 불확실성`은 많이 줄어들지만, 로드맵 기준으로는 보통 아직 아래가 남는다.

1. `scenario close` 공식 선언
2. `crop-group long-list(작물군 긴 후보 목록)` 생성
3. `crop x scenario hard filter(작물×시나리오 강한 탈락 필터)` 실행
4. survivor crop(살아남은 작물) 대상 `deep crop analysis(심층 작물 분석)`
5. 최종 `crop selection`

따라서 대답은 이거다.

- `아니, 이것만으로 자동으로 작물 선정까지 완료되진 않는다.`
- `하지만 지금 막혀 있는 핵심 병목은 거의 다 풀 수 있어서, 작물 선정 단계로 넘어가기 위한 마지막 큰 준비는 된다.`
