# 검증 컨텍스트 — MC1-1 하네스 v2.9

이 하네스에 대해 이미 알려진 사항:

1. **설계자:** Claude(Opus 4.6)가 설계하고 자체 품질 체크리스트로 검증함

2. **수정사항 A~E가 적용된 버전:**
   - **(A)** Stage 0에 불변 앵커 + 출처 규칙 삽입
   - **(B)** Stage 2에 게이트 조건 #4~#6 추가 (범위 준수, 완전성, 유전체 기술 타당성)
   - **(C)** Stage 0에 마이크로사이클 인식 앵커 추가
   - **(D)** 3-axis tag system — v2.3에서 도입, v2.4에서 정비, v2.5에서 enum 확장 완료
   - **(E)** Stage 2 검증 절차 구체화

3. **v2.4 P0 수정사항 (GPT-5.4 1차 외부 검증 반영):**

   - **P0-2:** 동적 시간 앵커 (`latest_available_year`)
   - **P0-3:** 3-axis tag system 일관성 정비
   - **P0-4:** Gate 기준 P0=100%, P1≥80% 분리
   - **P0-5:** D2/D4 NCBI/Ensembl 교차확인 + [미결] 태그

4. **v2.5 수정사항 (GPT-5.4 2차 외부 검증 반영):**
   - **P0-6:** MC1-1c 앵커 누락 해소 — 공통 앵커 블록 신설, 모든 sub-MC에 적용
   - **P0-7:** Few-shot 예시 연도 하드코딩 제거 — `{latest_available_year}` placeholder로 전환
   - **P0-8:** 3축 태그 enum 확장 — [문헌확인], [부분근거], [상충], [미결] 공식 추가
   - **P0-9:** 자동화 프로토콜 신설 — JSON wrapper schema, 상태전이 표, 체크포인트 문서 템플릿, 자동 루프 규칙

5. **v2.6 수정사항 (GPT-5.4 3차 외부 검증 반영):**

   - **P0-12:** FAIL retry target 충돌 해소 — 원인 기반 분기(E-2 전이표) 신설
   - **P1-1:** cross_verification 필수 필드화 + verdict 병합 규칙(F) 신설

6. **v2.7 수정사항 (GPT-5.4 4차 외부 검증 반영):**
   - **P0-13:** 자동 루프 기준을 `final_verdict` 유일 기준으로 통일 선언 (D. 자동 루프 규칙)
   - **P0-14:** conditional_detail.next_action enum에 `AUTO_REPAIR_AND_REJUDGE` 추가 + wrapper top-level `final_action` 필드
   - **P0-15:** E-2 rollback_state를 4개 상태 변수 전체로 완전 매핑

   - **P1-3:** E-2 FAIL 전이표에 cross_verification FAIL 행 추가

7. **v2.8 수정사항 (GPT-5.4 5차 외부 검증 반영):**
   - **P0-13 완결:** MC1-1b Gate판정에 gate.verdict 기반 문구 잔존 → 3개 Gate 모두 동일한 final_verdict 분기 템플릿으로 교체
   - **P0-17:** E-1 PASS 전이표에 `final_verdict` 적용 조건 명시 + Invariant 사후 검증 열 추가

8. **v2.9 수정사항 (GPT-5.4 6차 외부 검증 반영):**
   - **P0-19:** AUTO_REPAIR_AND_REJUDGE → final_action 매핑 명시 — wrapper schema 주석에 `final_action = PROCEED` 기록 규칙 추가. F 병합 규칙 뒤 **final_action 정제 규칙** 신설 (CONDITIONAL 행에서 E-3의 next_action이 final_action을 정제)
   - **P0-20:** DATA_SUFFICIENCY 전이 통일 — E-1 MC1-1a PASS 시 `DATA_SUFFICIENCY → PASSED` (기존 GATE_READY에서 변경). E-1 MC1-1b PASS에 `DATA_SUFFICIENCY → PASSED (유지)` 추가. E-2 MC1-1b retry 행에서 `GATE_READY (유지)` → `PASSED (유지)` 통일. B표 전이조건 정비
   - **P0-21:** human_judgment_required issue_type의 final_action override 규칙 — `cross_verification.issues`에 `human_judgment_required`가 1개라도 있으면 `final_action = HALT_HUMAN_CHECKPOINT`로 강제 override (F 병합 규칙의 기본 action보다 우선)
   - **P0-22:** DATA_SUFFICIENCY enum ordinal 정형화 — G. State Invariants에 `INCOMPLETE(0) < GATE_READY(1) < PASSED(2)` 명시. 교차 변수 조건을 `DATA_SUFFICIENCY = PASSED` (ordinal ≥ 2) 기준으로 업데이트

10. **구조적 한계:** "AI가 만든 검증 체계를 AI가 적용"하는 자기참조 구조이므로, 독립적인 외부 모델(GPT)에 의한 교차검증이 필요함

11. **외부 검증 이력:**
    - **1차 (v2.3):** GPT-5.4 Thinking → CONDITIONAL PASS. P0 5건, P1 다수.
    - **2차 (v2.4):** GPT-5.4 Thinking → FAIL. MC1-1c 앵커 누락, Few-shot 연도 하드코딩. → v2.5에서 해소.
    - **3차 (v2.5):** GPT-5.4 Pro → FAIL. 3축 FAIL (조건부게이트, 실행가능성, MC대응). → v2.6에서 해소.
    - **4차 (v2.6):** GPT-5.4 Pro → FAIL. 3축 FAIL (스키마연속성, 실행가능성, MC대응). → v2.7에서 해소.
    - **5차 (v2.7):** GPT-5.4 Pro → FAIL. 3축 FAIL. 핵심: gate.verdict 잔존, invariant 부재. → v2.8에서 해소.
    - **6차 (v2.8):** GPT-5.4 Pro → FAIL. 4축 FAIL (스키마연속성, 조건부게이트, 실행가능성, MC대응). 핵심: (1) AUTO_REPAIR→final_action 매핑 미비, (2) DATA_SUFFICIENCY 전이 비일관 (E-1/E-2/B표 간 GATE_READY vs PASSED 혼재), (3) human_judgment_required가 F 병합 규칙의 RETRY와 충돌, (4) DATA_SUFFICIENCY enum ordinal 미정의로 ≥ 비교 불가.
