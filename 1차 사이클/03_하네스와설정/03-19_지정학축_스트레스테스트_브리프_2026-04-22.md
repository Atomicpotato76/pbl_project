# 2026-04-22 지정학 구조 스트레스 테스트 브리프

## 목적

이 문서는 현재 종자선정 패키지에 빠져 있던 `지정학적 층위`를 외부 모델이 일관되게 감사할 수 있도록 만든 보조 브리프다.

이번 검증의 목적은 국제정세 일반론을 늘어놓는 것이 아니다.
목적은 `이 작물이 이 시나리오-국가-기관 경로에서 실제로 흡수될 수 있는가`를 구조적으로 따지는 것이다.

즉, 이 층위는 단순한 배경설명이 아니라 아래 질문을 겨냥한다.

- 현지에서 실제로 먹는가, 쓰는가, 기르는가
- 국가 단위 말고 목표 지역/제도/구매자 단위에서도 성립하는가
- 정책 문구가 아니라 실제 buyer / payer / seed-refresh 경로가 있는가
- 무역, 가격, 제도 전환, ABS/거버넌스 충격이 경로를 깨지 않는가

## 왜 이 층위가 필요한가

현재 패키지는 `scenario fit`, `genome necessity`, `delivery route`를 상당히 많이 다뤘지만, 그 사이를 연결하는 `지정학적 경로 적합성`은 충분히 분리하지 못했다.

그 결과 아래 같은 오류가 날 수 있다.

1. `인도는 펄스를 많이 먹는다`를 `이 시나리오에서 lentil 또는 urdbean이 실제로 맞다`로 과잉 일반화
2. `공공 프로그램이 pulses를 말한다`를 `우리 작물과 exact buyer path가 잠겼다`로 과잉 해석
3. `작물 자체는 좋다`를 `종자/유전체 프로젝트로도 실행된다`로 혼동
4. `한 국가 안의 일부 지역/기관 적합성`을 `전국 단위 적합성`으로 잘못 확장

즉 이 층위는 "좋은 작물인가"가 아니라 "이 경로에서 굴러가는 작물인가"를 보는 층위다.

## 새 지정학 축

아래 7개 축을 이번 스트레스 테스트의 기본 축으로 둔다.

### G1. 최종 사용 적합성

핵심 질문:
- 이 작물이 목표 국가와 목표 지역에서 실제 식문화, 사료체계, 가공관행 안에 들어가는가
- "국가 전체에서 알려진 작물"과 "우리가 겨냥하는 사용처에서 실제로 선택되는 작물"을 구분했는가

왜 중요한가:
- 이 축이 비면 "좋은 작물인데 아무도 안 먹음 / 안 씀 / 안 삼"이 된다.

기본 판정:
- `default gate`

### G2. buyer / payer specificity

핵심 질문:
- 누가 실제 구매자인가
- 공공조달, 학교급식, buffer, 사료기업, 협동조합, seed company, FPO 등 구체 주체가 있는가
- 제도가 crop-neutral인지 crop-specific인지 구분했는가

왜 중요한가:
- "수요는 있다"와 "우리 경로에 buyer가 있다"는 다르다.

기본 판정:
- `default gate`

### G3. 지역 이질성과 집중도

핵심 질문:
- 국가 단위 평균이 지역별 분산을 가리고 있지 않은가
- 특정 주/도/카운티/생태권에서만 성립하는가
- 만약 그렇다면 그 제한이 시나리오에 치명적인가

왜 중요한가:
- 인도 같은 경우 national fit만 보면 오판하기 쉽다.

기본 판정:
- `weight`, 단 심한 mismatch면 `conditional gate`

### G4. 무역/수입/가격 체제 노출도

핵심 질문:
- 이 작물이 수입 의존, 관세, buffer stock, MSP/PSS, 국제가격 변동, 외교 갈등, 공급망 충격에 얼마나 좌우되는가
- 국내 생산 장려와 실제 시장가격 유인이 서로 충돌하지 않는가

왜 중요한가:
- 제도는 있어도 가격체제가 경로를 흔들면 실제 adoption은 무너진다.

기본 판정:
- `weight`, 단 정책-시장 충돌이 심하면 `conditional gate`

### G5. 종자 채널 / 채택 / 가치회수

핵심 질문:
- 누가 종자를 갱신하고, 누가 가치를 회수하며, 왜 새 유전체 자산이 현장 결정을 바꿀 것인가
- farm-saved seed, public seed, cooperative seed, private seed가 어떻게 섞이는가

왜 중요한가:
- buyer가 있어도 seed-refresh와 value capture가 없으면 프로젝트 경로가 약하다.

기본 판정:
- `weight`, 단 seed route 부재가 심하면 `conditional gate`

### G6. 규제 / 주권 / ABS / 거버넌스

핵심 질문:
- 국경 간 유전자원, DSI, ABS, source holder, MTA/SMTA, 품종권/종자법, 공공기관 협력 규정이 경로를 막지 않는가
- 공식 법령과 실제 운영 포털/기관 관행 사이 충돌이 있는가

왜 중요한가:
- 실행 경로가 제도적으로 막히면 작물 적합성과 무관하게 탈락한다.

기본 판정:
- `default gate`

### G7. 정권/정책 지속성과 전략적 내구성

핵심 질문:
- 이 경로가 3~5년 시야에서 유지될 가능성이 있는가
- 일시 정책인지, 구조적 프로그램인지, 정권 변화나 예산 축소에 취약한가

왜 중요한가:
- 현재 맞는 경로가 다음 작기나 다음 회계연도에 바로 사라지면 프로젝트 안정성이 낮다.

기본 판정:
- `weight`, 단 특정 단일 프로그램 의존이면 `conditional gate`

## 이번 검증에서 반드시 구분해야 할 것

아래 혼동은 금지한다.

1. `국가 차원 수요`와 `우리 작물의 exact route fit`
2. `crop-neutral policy`와 `crop-specific buyer lock`
3. `작물 존재감`과 `종자/유전체 프로젝트 실행성`
4. `공식 문구`와 `실무 작동 경로`
5. `species-level plausibility`와 `scenario-level absorbability`

## gate vs weight 기본 규칙

이번 브리프의 기본값은 아래와 같다.

- 기본 gate: `G1`, `G2`, `G6`
- 기본 weight: `G3`, `G4`, `G5`, `G7`

단, 외부 모델이 강한 근거를 제시하면 일부 축은 `conditional gate`로 승격할 수 있다.

예:
- 지역 mismatch가 치명적이면 `G3 -> conditional gate`
- 가격/수입체제가 경로를 거의 무효화하면 `G4 -> conditional gate`
- seed/value-capture 부재로 프로젝트 자체가 성립하지 않으면 `G5 -> conditional gate`
- 단일 정책 의존이 너무 크면 `G7 -> conditional gate`

## 현재 패키지에 적용할 때의 초점

### N1

- 국가/경로: Kenya forage / source-holder / ABS / public-private route
- 특별히 민감한 축:
  - `G2 buyer / payer specificity`
  - `G6 규제 / ABS / 거버넌스`
  - `G7 정책 지속성`

### N2

- 국가/경로: India pulse / protein / public demand / procurement / seed-refresh route
- 특별히 민감한 축:
  - `G1 최종 사용 적합성`
  - `G2 buyer / payer specificity`
  - `G3 지역 이질성`
  - `G4 무역/가격 체제`
  - `G5 종자 채널 / 가치회수`

## 외부 감사자에게 던질 핵심 질문

1. 위 7개 축이 충분한가, 아니면 ranking-critical한 축이 빠졌는가
2. 어떤 축은 사실 `gate`로 봐야 하는데 우리가 `weight`처럼 다루고 있는가
3. 현재 N1/N2 provisional lead는 이 축을 넣어도 유지 가능한가
4. 어떤 결론은 추가 search로 해결 가능한 `search debt`이고, 어떤 것은 조사 후에도 남는 `investigated_unknown`인가
5. "인도는 pulses를 먹는다" 같은 거친 국가 단위 서술이 실제 crop choice를 과잉 단순화하고 있지 않은가

## 금지할 응답 패턴

- 거시 국제정세 일반론만 길게 쓰기
- 국가 소개나 농업개론으로 분량 채우기
- 현재 ranking을 새로 뽑아버리기
- search debt를 곧바로 부정결론으로 번역하기
- buyer path 없이 "수요가 있다"만 반복하기

## 기대하는 결과

외부 모델은 아래를 분리해서 말해야 한다.

1. `지정학 구조 자체의 설계 품질`
2. `N1 / N2 각각에서 어느 축이 실제 blocker인지`
3. `추가 search로 잠글 수 있는 부분`
4. `조사해도 당분간 불확실성으로 남을 genuine unknown`

이 네 가지가 분리되지 않으면 이번 스트레스 테스트는 다시 narrative 요약으로 흘러갈 위험이 있다.
