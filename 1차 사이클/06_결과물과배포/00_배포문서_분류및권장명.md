# 배포문서 분류 및 실제 적용명

이 문서는 `03_배포세트` 안에 있는 문서를 **구 파일 / 신 파일 / 공통 입력 / 참고보관**으로 나누고,
형주가 바로 알아볼 수 있게 **진행 순서 번호 + 한글 통일명**으로 다시 읽는 기준을 만든 지도다.

중요:

- 이 문서는 **2026-04-22 기준 실제 rename이 적용된 상태**를 반영한다.
- 아래 표에서 `기존` 열은 정리 전 이름이고, `현재` 열은 지금 패키지에서 실제로 쓰는 이름이다.
- 배포세트 바깥 문서 중 역사 서술은 일부 옛 이름을 유지할 수 있으므로, 실제 탐색은 항상 `현재 이름` 기준으로 본다.

---

## 1. 먼저 결론

`03_배포세트` 안 문서는 크게 4종류다.

1. `공통 입력자료`
   문제 정의, 로드맵처럼 구/신 체계 모두가 같이 쓰는 문서
2. `구 파일`
   초기 작물선정 중심 판단 체계와 그 결과
3. `신 파일`
   시나리오 우선 판단 체계와 그 검증 흐름
4. `참고보관`
   현재 직접 쓰기보다 과거 구조를 복기할 때 보는 문서

그리고 형주가 헷갈렸던 포인트 하나를 먼저 말하면:

- `03_배포세트` 안에는 **하네스 원문이 없다**
- `03_배포세트` 안에는 **피드백 로그도 없다**
- 실제로 들어 있는 건 mostly
  - 입력자료
  - 판단 기준표
  - 검증 프롬프트
  - 조사 결과
  - 판정 결과

즉, `하네스`, `피드백`, `현재 활성 기준본`까지 한 폴더에 다 있는 구조가 아니라,
배포용 세트 안에 일부만 복제돼 있어서 더 헷갈렸던 거다.

## 2. 권장 번호 체계

앞으로 `03_배포세트` 안 문서를 읽거나 이름 붙일 때는 아래 숫자 체계를 쓰면 된다.

- `00` 공통 입력
- `10` 구기준
- `11` 구결과
- `12` 전환자료
- `20` 신원칙
- `21` 신기준
- `30` 검증대상 정리
- `40` 검증실행 프롬프트
- `50` 조사결과
- `60` 판정결과
- `90` 참고보관

## 6. 형주 기준으로 헷갈리는 포인트 바로 정리

### 6-1. `최종_시나리오선정기준`은 왜 3개냐

- `v1` = 초안
- `v1.1` = 수정본
- `v1.2` = 강화본

즉 앞으로는 이렇게 읽으면 된다.

- `v1`, `v1.1` = 구 파일
- `v1.2` = 신 파일

### 6-2. `스크리닝결과_v3`는 어디에 속하냐

이건 이름만 보면 최신 같지만, 실제 역할은 `구 체계의 결과물`이다.
즉 `작물 먼저 뽑고 점수화하던 흐름`의 결과이므로 `구결과`로 읽는 게 맞다.

### 6-3. `시나리오_시드_및_실패사례_v1`는 구냐 신이냐

완전히 신 파일이라기보다 `전환자료`다.
구 체계의 한계를 드러내면서 신 체계로 넘어가게 만든 문서라,
별도 분류로 두는 게 덜 헷갈린다.

### 6-4. 배포패키지 안에 없는 문서

배포패키지 안에는 아래가 없다.

- 현재 작물선정 하네스 원문
- 하네스 검증 프롬프트
- 검증 컨텍스트
- 피드백 로그/사후감사 문서

이건 `최신 상황`이나 `기록`에서 봐야 한다.

## 8. 적용 완료 메모

이번 정리에서 실제로 적용된 건 아래다.

1. 폴더 이름을 번호 체계로 바꿨다.
2. 각 세트 안 파일명을 위 표의 `현재 파일명` 기준으로 통일했다.
3. 패키지 안 핵심 안내문과 주요 참조 문서의 명시적 경로를 새 이름으로 갱신했다.
4. 역사 설명 문서에는 옛 이름이 일부 남을 수 있지만, 실제 파일 탐색과 첨부는 새 번호 이름을 기준으로 하면 된다.
