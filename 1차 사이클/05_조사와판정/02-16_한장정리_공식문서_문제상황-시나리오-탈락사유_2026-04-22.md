# 2026-04-22 한 장 정리 — 공식 문서

## 1. 문제상황

본 프로젝트의 직접 과제는 `최근 국내외에서 수요가 증가하는 작물`을 대상으로 `표준유전체(reference genome, 기준이 되는 대표 유전체)` 연구 프로젝트를 5주 안에 기획하는 것이다.  
따라서 현재 작물 선정 단계의 판정 기준은 단순 시장성 평가가 아니라, `수요 증가`, `reference genome 필요성`, `보급·회수 경로`, `법·제도 적합성`, `5주 기획 가능성`의 동시 충족 여부다.

현 단계는 `최종 작물 확정`이 아니라 `scenario close 직전의 검증 단계`이며, 모든 결론은 `working / provisional selection`으로 관리한다.

---

## 2. 판단 체크리스트와 근거

### 2-1. 시나리오 선정 체크리스트

시나리오 판정은 `SG-1~SG-8`을 사용한다.

| 코드 | 판단축 | 판정 의미 |
|---|---|---|
| `SG-1` | PBL 적합성 | 수요 증가 작물의 reference genome 기획으로 수렴하는가 |
| `SG-2` | 수요 증가 및 유형 명확성 | 증가가 실재하며 소비·생산·수입·조달 유형이 구분되는가 |
| `SG-3` | buyer/payer 분리 | beneficiary, buyer, payer, decision-maker가 분리되는가 |
| `SG-4` | 사업·공공 경로 | commodity demand가 seed/genomics route 또는 공공 route로 이어지는가 |
| `SG-5` | delivery 명시성 | named actor 기준 전달 경로가 있는가 |
| `SG-6` | 법·제도 경로 | ABS, seed law, farmers’ rights 등을 공식 자료로 설명 가능한가 |
| `SG-7` | genome necessity | 새 표준유전체가 없으면 불리해지는 핵심 use-case가 있는가 |
| `SG-8` | heroic assumption 차단 | “대기업이면 가능” 같은 과도한 가정이 baseline에 숨어 있지 않은가 |

### 2-2. 작물 필터 체크리스트

작물 판정은 `HF-0~HF-9`를 사용한다.  
그중 현재 가장 강한 fail-fast는 `HF-0 공개 유전체 충분성`이다.

| 코드 | 판단축 | 핵심 분기 |
|---|---|---|
| `HF-0` | 공개 유전체 충분성 | 기존 공개 genome/pangenome/resource로 이미 충분하면 즉시 불리 |
| `HF-1` | 시나리오 적합성 | 작물이 해당 시나리오에 자연스럽게 들어가는가 |
| `HF-3` | novelty | 기존 자원 대비 추가 가치가 남는가 |
| `HF-4` | genome-to-breeding bridge | genome이 실제 육종 의사결정으로 이어지는가 |
| `HF-5` | legal access | germplasm access와 법 경로가 공식 자료로 설명되는가 |
| `HF-7` | technical feasibility | genome size, ploidy, assembly 부담을 감당 가능한가 |
| `HF-9` | geopolitical kill-switch | 즉시 붕괴성 리스크가 있는가 |

### 2-3. 주요 근거군

- `PBL 문제상황`
- `시나리오 선정 기준 v1.2`
- `작물선정 하네스 v2.9`
- 국가·정부·공식기관 자료
- NCBI, PubMed, Genome Warehouse 등 공개 유전체 자원 현황 자료

## 3. 조사 하네스

조사 하네스(harness, 조사 절차 규칙집)는 아래 순서로 운용한다.

1. crop-agnostic scenario 설계
2. `SG-1~SG-8` 적용
3. 통과 시나리오에만 crop long list 부착
4. `HF-0` fail-fast prescreen
5. `HF-1~HF-9` 적용
6. 결과를 `carry / watch / drop`으로만 관리
7. `HF-0`, `HF-3`, `HF-5`, `HF-7` 미해결 항목을 carry-forward

즉, 현재 단계의 목적은 `최종 추천`이 아니라 `다음 검증 단계로 넘길 자격 판정`이다.

## 4. 시나리오 판정 현황

| 시나리오 | 현재 상태 | 판정 요약 |
|---|---|---|
| `N1` 케냐 낙농 사료작물 회복력 경로 | `active` | genome necessity와 formal seed route가 가장 강함 |
| `N2` 인도 공공 단백질-식량안보 식용 콩과 경로 | `active` | buyer/payer와 공공 route 설명이 가장 선명함 |
| `P1` 페루 공공·혼합 영양 경로 | `inactive` | buyer story는 있으나 exact crop 단계에서 `HF-0/HF-3/SG-6/SG-7` 리스크가 큼 |
| `Brazil` 가족농 공공조달 경로 | `reserve` | 제도선은 강하나 crop-level novelty가 약해질 수 있음 |
| `Morocco` 수자원 압박 적응 경로 | `reserve` | climate/public route는 강하나 crop-level genome moat가 얇음 |

## 5. 시나리오별 작물 스택과 탈락 사유

| 시나리오 | `carry` | `watch` | `drop` | 주된 탈락 사유 |
|---|---|---|---|---|
| `N1` | `Chloris gayana`, `Chloris roxburghiana` | `Cenchrus ciliaris` | `Napier`, `Urochloa/Brachiaria` | `Napier`, `Urochloa`는 `HF-0/HF-3`에서 공개 자원 포화도가 높음. `Cenchrus`는 `HF-7` 기술 난도 부담 |
| `N2` | `lentil`, `urdbean` | `mungbean`, `field pea` | `chickpea`, `pigeonpea`, `horsegram`, `ricebean`, `grass pea` | `chickpea`, `pigeonpea`는 공개 genome과 breeding resource가 과도하게 성숙. `horsegram`, `ricebean`은 route weakness. `grass pea`는 규제 blocker |
| `P1` | `kiwicha`, `cañihua` | `tarwi` | `quinoa`, `maca` | `quinoa`는 genome saturation, `maca`는 school-feeding grain route와의 fit 약화 |
| `Brazil` | `cowpea` | `sweet potato`, `cassava` | `common bean`, `maize` | `common bean`, `maize`는 `HF-0/HF-3` very high risk |
| `Morocco` | `lentil` | `faba bean`, `barley` | `durum wheat`, `chickpea` | `wheat`, `chickpea`는 saturation이 너무 강하고, `faba bean`은 technical burden이 큼 |

## 6. 현재 운영 결론

- 현재 mainline `working selection`: `N1 x Chloris gayana`
- 현재 backup `working selection`: `N2 x lentil`
- 둘 다 `final crop selection`이 아니라 `conditional live lane`이다.
- 다음 이동 조건은 `HF-0 direct audit`, `source holder`, `ABS/legal route`, `named buyer/delivery path`를 더 잠그는 것이다.

따라서 현재 프로젝트의 공식 해석은 아래와 같다.

> `scenario-first` 구조 아래에서 `N1`, `N2`만 active로 유지하고, 각 시나리오 안에서는 공개 유전체 포화도와 route clarity를 기준으로 작물을 `carry / watch / drop`으로 관리한다. 아직 최종 확정 단계는 아니다.
