# 2026-04-22 선정 작물 확정 이후 표준유전체 구축 및 활용 후속 로드맵

## 문서 목적

이 문서는 현재 프로젝트가 `시나리오 잠금 직전`에서 멈춰 있는 상태를 전제로,  
`선택 작물이 확정된 직후`부터 어떤 순서로 `표준유전체 구축 기획`과 `향후 활용 설계`를 이어 가야 하는지 정리한 후속 로드맵이다.

핵심 목적은 세 가지다.

1. 현재 `MC1-1`의 종료 조건을 명확히 한다.
2. 작물이 확정되면 곧바로 들어갈 `MC2 이후 세부 로드맵`을 제공한다.
3. 발표용 기획을 넘어서, 실제 연구개발 관점에서 `표준유전체 → 육종 활용`으로 이어지는 경로를 한 번에 보여 준다.

---

## 1. 현재 상황과 진행도

로컬 최신 문서 기준 현재 상태는 아래와 같다.

- 현재 상위 위치: `MC1-1 내부`
- 현재 세부 위치: `시나리오 close 직전 마지막 lock cycle`
- 활성 시나리오: `N1`, `N2`
- 주력 조합: `N1 x Chloris gayana`
- 백업 조합: `N2 x Lens culinaris`
- 현재 상태값: `accept_for_crop_selection = none`, `ready_to_move = no`

즉, 현재 프로젝트는 이미 `작물 선정 논리의 재설계`까지는 끝냈지만,  
아직 `최종 작물 1개를 공식 확정하고 MC2로 넘어가는 gate`는 통과하지 못한 상태다.

현재까지 끝난 것과 아직 안 끝난 것을 분리하면 아래와 같다.

### 1-1. 이미 끝난 것

- `crop-first`에서 `scenario-first`로 운영 원리 전환
- broad lane screening 종료
- narrow scenario shortlist 종료
- active stack을 `N1/N2 only`로 잠금
- `HF-0 public genome sufficiency`를 최상위 필터로 두는 원칙 정리
- 법·제도, buyer/payer/decision-maker 분리 원칙 정리

### 1-2. 아직 안 끝난 것

- `N1` 또는 `N2`를 crop selection으로 넘겨도 될 만큼 scenario close가 충분한지 최종 판단
- exact crop와 exact source holder의 연결 잠금
- `HF-0` 직접 심사
- `왜 새 표준유전체인가`를 작물 수준에서 한 문장으로 잠그는 일

따라서 현재 병목은 `조사를 더 하느냐`가 아니라,  
`다음 단계로 넘어가도 되는 정도로 논리와 근거를 잠그느냐`에 있다.

## 2. MC1-1 종료 조건

선택 작물 확정은 단순히 후보 중 하나를 고르는 행위가 아니다.  
`이 작물에 대해 이후 모든 유전체 설계와 활용 논리가 흔들리지 않을 만큼 프로젝트 thesis를 잠그는 일`이다.

`MC1-1 종료`는 아래 다섯 산출물이 모두 채워졌을 때로 정의하는 것이 가장 안전하다.

### 2-1. 종료에 필요한 최소 산출물

1. `최종 작물 카드`
   - 학명, 공통명, 기준 시나리오, 최종 선택 이유 3줄

2. `genome necessity memo`
   - 기존 공개 reference/draft/pangenome 자원 현황
   - 왜 기존 공개 자원만으로는 이번 과제의 질문이 충분히 해결되지 않는지

3. `source-holder/legal memo`
   - source holder
   - 접근 경로
   - ABS 또는 breeder-derived carve-out 판단
   - 데이터 공개 시 제약 여부

4. `reference thesis statement`
   - 예: `이 과제의 핵심 산출물은 [작물명]의 breeding-grade chromosome-level reference genome 설계안이다`
   - 또는 `이미 공개 reference가 존재하므로, local elite line 기반 reference upgrade + pangenome starter를 목표로 한다`

5. `next-step handoff packet`
   - MC2-1에서 바로 사용할 입력값 묶음
   - genome size, ploidy, known assembly, candidate line/accession, 핵심 형질, 활용 시나리오

### 2-2. 종료 판정 규칙

아래 중 하나라도 남아 있으면 MC1-1은 닫지 않는 것이 좋다.

- `HF-0`가 아직 `UNKNOWN`
- legal route가 `추정`
- source holder가 `기관/회사 이름 수준`으로 고정되지 않음
- 선택 이유가 `시장성은 좋은데 왜 새 reference인가` 질문에 답하지 못함

즉, MC1-1의 진짜 gate는 `작물 확정`이 아니라 `작물 thesis 확정`이다.

## 3. 선택 작물 확정 직후의 후속 로드맵

선택 작물이 확정되면 이후 로드맵은 `유전체 상태 복원 → reference build 전략 설계 → annotation/QC → 활용 설계` 순서로 가는 것이 가장 자연스럽다.

현재 문서 체계에 맞추면 아래처럼 확장하는 것이 좋다.

| 단계 | 제안 코드 | 핵심 질문 | 산출물 |
|---|---|---|---|
| 0 | `MC1-1 close` | 왜 이 작물인가, 왜 새 reference인가 | 최종 작물 카드, necessity memo |
| 1 | `MC2-1` | 이 작물의 유전체는 어떻게 생겼고 기존 자원 공백은 어디인가 | genome status audit |
| 2 | `MC2-2` | 어떤 라인과 어떤 데이터 조합으로 reference를 설계할 것인가 | sequencing + assembly design |
| 3 | `MC2-3` | 어떻게 QC, annotation, 공개를 설계할 것인가 | QC/annotation/release plan |
| 4 | `MC2-4` | 이 reference를 실제 육종/종자 전략에 어떻게 연결할 것인가 | utilization roadmap |
| 5 | `MC2-5` | 발표와 실행 계획을 어떻게 하나의 narrative로 묶을 것인가 | final synthesis |

아래는 각 단계를 더 세부적으로 풀어 쓴 버전이다.

## 4. 단계별 세부 로드맵

## 4-1. 단계 0 — 선택 작물 thesis 잠금

### 목적

작물을 고르는 순간 바로 `reference genome 과제의 형태`를 동시에 고정한다.

### 해야 할 일

- 선택 작물 1개와 backup 1개 확정
- 공개 자원 현황을 빠르게 audit
- 프로젝트 유형을 아래 셋 중 하나로 분류
  - `A형`: 아직 breeding-grade reference가 약함 → de novo chromosome-level reference 중심
  - `B형`: draft는 있으나 품질이 낮음 → reference upgrade 중심
  - `C형`: chromosome-level reference가 이미 있음 → pangenome/elite-line reference/trait-genome 중심

### 단계 종료 산출물

- `project type` 분류표
- `왜 first-reference인가 / 왜 upgrade인가 / 왜 pangenome-first인가` 1문단

---

## 4-2. 단계 1 — Genome Status Audit (`MC2-1`)

### 목적

선정 작물의 유전체 구조와 기존 공개 자원 수준을 객관적으로 복원한다.

### 핵심 질문

- genome size는 얼마인가
- diploid/polyploid 중 어느 구조인가
- heterozygosity와 repeat burden은 어느 정도인가
- 기존 assembly는 draft인지 chromosome-level인지
- pangenome 또는 variation catalog가 이미 있는가
- 이번 프로젝트가 개선해야 할 공백은 정확히 무엇인가

### 조사 항목

- genome size, chromosome number, ploidy
- repeat content, heterozygosity
- 기존 reference의 assembly level, contiguity, completeness
- annotation 존재 여부와 품질
- 기 확보된 resequencing/pangenome/SV 자원
- 근연종 reference와 synteny 활용 가능성

### 단계 종료 기준

아래 네 문장이 자연스럽게 이어지면 다음 단계로 간다.

1. `이 작물의 유전체 구조는 이렇다`
2. `현재 공개 reference 자원은 여기까지 와 있다`
3. `이번 프로젝트는 이 공백을 메운다`
4. `그래서 필요한 데이터 조합은 이 방향이다`

---

## 4-3. 단계 2 — Reference Build Strategy (`MC2-2`)

### 목적

실제 reference genome을 어떤 샘플과 어떤 시퀀싱 조합으로 구축할지 설계한다.

### 먼저 잠가야 할 것

- reference accession 또는 cultivar 1개
- 왜 이 line이 reference에 적합한가
- heterozygosity를 낮출 수 있는가
- DNA 추출 가능성과 seed access가 현실적인가

### 주요 설계 항목

1. `reference line 선정`
   - inbred line, landrace, released variety, breeder line 중 선택
   - 선택 기준: 대표성, 순도, seed access, downstream utility

2. `sequencing recipe`
   - 기본 reference 목표: HiFi 중심 + Hi-C
   - 복잡 유전체 또는 T2T ambition: HiFi + ultra-long ONT + Hi-C
   - 필요 시 short-read는 polishing 또는 panel genotyping 용도로만 보조

3. `assembly architecture`
   - primary assembly
   - haplotype resolution 여부
   - scaffolding
   - polishing
   - contamination/organelle separation

4. `success metric`
   - chromosome assignment
   - k-mer completeness
   - single-copy gene completeness
   - false duplication 관리

### 단계 종료 산출물

- reference line card
- sequencing design 2안 비교표
- 추천 조합 1안
- assembly flowchart
- 예상 비용/일정 범위

---

## 4-4. 단계 3 — QC, Annotation, Release (`MC2-3`)

### 목적

reference를 `읽었다`에서 끝내지 않고, `쓸 수 있는 기준 유전체`로 만든다.

### QC에서 반드시 분리해야 할 축

- assembly quality
- annotation quality
- biological validity
- public release readiness

### QC 체크리스트

- contamination screening
- organelle 분리
- primary assembly와 alternate haplotype bin 구분
- BUSCO 등 보존유전자 completeness
- k-mer completeness / consensus quality
- chromosome anchoring 수준

### annotation 설계

- repeat annotation 먼저 수행
- transcript evidence와 protein homology evidence 결합
- protein-coding gene, ncRNA, organellar annotation 분리
- trait-relevant gene family는 수동 검토 후보로 따로 표시

### release 설계

- raw reads archive
- assembly accession 등록
- annotation 파일 공개
- metadata, BioProject, BioSample 정리
- versioning 규칙 정의

### 단계 종료 산출물

- QC dashboard 정의
- annotation workflow
- public submission checklist
- version 1.0 release plan

---

## 4-5. 단계 4 — 활용 설계 (`MC2-4`)

### 목적

reference genome을 `논문용 자원`이 아니라 `육종·종자 의사결정 자산`으로 전환한다.

### 활용 경로는 최소 네 갈래로 정리하는 것이 좋다

1. `variant discovery`
   - SNP, indel, SV catalog 구축

2. `trait genetics`
   - QTL, GWAS, haplotype block, candidate gene mapping

3. `breeding deployment`
   - marker development
   - parental selection
   - seed purity / identity QC
   - genomic selection 학습세트 구축

4. `germplasm strategy`
   - core collection 설계
   - wild relative / landrace 도입
   - 이후 pangenome 확장

### 활용 질문 예시

- 어떤 형질을 가장 먼저 reference와 연결할 것인가
- 현장 검증 가능한 phenotype은 무엇인가
- marker로 바로 전환 가능한 locus가 있는가
- seed company/public breeding route에 어떤 의사결정 지점을 제공하는가

### 단계 종료 산출물

- trait-to-genome map
- marker deployment concept note
- breeding decision use-case 3개
- phase 2 pangenome 확장 제안

---

## 4-6. 단계 5 — 발표/실행 통합 (`MC2-5`)

### 목적

위의 모든 산출물을 하나의 내러티브로 묶는다.

### 기본 연결 문장

`왜 이 작물인가 → 왜 기존 자원으로 부족한가 → 어떤 reference를 만들 것인가 → 어떻게 품질을 담보할 것인가 → 그래서 무엇을 바꿀 수 있는가`

### 최종 산출물

- 발표 초안
- 실행 요약표
- 리스크와 fallback
- 후속 연구 제안

## 5. 실제 연구개발 기준의 장기 로드맵

위 4장은 `기획 과제용 로드맵`이다.  
실제 연구개발 프로젝트로 연결하면 아래처럼 12-24개월 구조로 보는 것이 좋다.

| 기간 | 실제 R&D 단계 | 목표 | 대표 산출물 |
|---|---|---|---|
| 0-1개월 | target lock | accession, legal route, phenotype thesis 확정 | sample/legal/go-no-go pack |
| 1-3개월 | reference generation | long-read + Hi-C 기반 assembly 생성 | v0.9 assembly |
| 3-5개월 | curation/QC | misjoin 수정, contamination 제거, chromosome anchoring | v1.0 reference |
| 4-6개월 | annotation | repeat + transcript + protein evidence 기반 annotation | v1.0 annotation |
| 6-12개월 | diversity panel | 20-200 accession resequencing/genotyping | SNP/SV catalog |
| 9-15개월 | trait mapping | QTL/GWAS/haplotype 분석 | marker 후보 |
| 12-24개월 | deployment | marker-assisted selection, parental choice, seed QC | breeding toolkit |

핵심은 `reference build`가 끝이라고 보지 않고,  
그 뒤의 `variation layer`와 `trait layer`를 반드시 붙여야 실사용 가치가 생긴다는 점이다.

## 6. 작물 확정 후 분기 규칙

선택 작물이 무엇이냐에 따라 후속 로드맵의 thesis가 달라진다.

## 6-1. 저포화 작물일 때

예: `Chloris gayana`처럼 draft resource는 있으나 breeding-grade chromosome-scale 자원이 약한 경우

이 경우의 주 thesis는 아래가 된다.

- `new or upgraded chromosome-level reference`
- `formal seed route와 연결된 reference line`
- `후속 diversity panel과 marker 개발의 기준 좌표 확보`

즉, 목표는 `reference를 만드는 것 자체`가 아직 주요 성과가 된다.

## 6-2. 중간 포화 작물일 때

예: `Vigna mungo`처럼 long-read draft는 있으나 더 강한 reference와 현장 활용 연결이 필요한 경우

이 경우의 주 thesis는 아래가 더 적합하다.

- `draft-to-chromosome upgrade`
- `India/public breeder relevant elite line reference`
- `trait-oriented annotation and variation catalog`

즉, 목표는 `첫 reference`보다 `실사용 가능한 upgrade`에 있다.

## 6-3. 고포화 작물일 때

예: `Lens culinaris`처럼 chromosome-level assembly가 이미 공개된 경우

이 경우 `새 reference genome을 또 하나 만든다`는 주장만으로는 약할 수 있다.  
따라서 thesis를 아래처럼 바꾸는 것이 맞다.

- `local elite line reference`
- `stress/quality/public-breeding pangenome starter`
- `graph reference 또는 haplotype-aware breeding resource`

즉, 이 경우의 핵심은 `reference 단일본`이 아니라  
`기존 reference를 넘어서 breeding decision에 더 직접적으로 쓰이는 variation-aware resource`다.

## 6-4. 현재 유력 후보에 대한 외부확인 메모

후속 로드맵을 실제로 쓸 때는 아래처럼 `작물별 공개 자원 상태`를 먼저 확인하고 thesis를 정하는 것이 안전하다.

### `Chloris gayana`

- 2023년에 diploid Rhodes grass의 `첫 annotated draft genome`이 보고되었다.
- 따라서 thesis를 `first-ever genome`으로 두기보다, `chromosome-level upgrade` 또는 `breeding-grade reference`로 두는 편이 더 정확하다.

### `Lens culinaris`

- 2025년에 chromosome-level lentil assembly가 공개되어 있다.
- 따라서 이 작물이 최종 선택되면, thesis는 `새 단일 reference`보다 `local elite line`, `stress/quality-oriented pangenome starter`, `public breeding resource` 쪽으로 이동하는 것이 더 자연스럽다.

### `Vigna mungo`

- 2022년에 long-read 기반 blackgram draft genome이 공개되어 있다.
- 따라서 이 경우도 `완전 무자원 작물` 프레이밍보다는 `draft-to-chromosome upgrade`와 `trait-linked breeding resource` 프레이밍이 더 적합하다.

## 7. 외부 기준을 반영한 품질 목표

2026년 4월 현재 외부 기준을 반영하면, 최소한 아래 정도의 품질 목표를 로드맵에 넣는 것이 좋다.

### 7-1. 최소 reference 품질 목표

- `6.C.Q40` 수준 또는 그에 준하는 chromosome-scale reference를 목표로 둔다.
- 가능한 경우:
  - `>90%` k-mer completeness
  - `>90%` sequence의 chromosome assignment
  - `>90%` single-copy conserved genes complete and single copy
  - `<5%` false duplication

### 7-2. 데이터 조합의 기본 방향

- 기본 reference build:
  - `PacBio HiFi 중심 + Hi-C`
- 복잡 유전체 또는 T2T 지향:
  - `고심도 HiFi + ultra-long ONT + Hi-C`

### 7-3. 공개와 제출 기본 원칙

- raw reads와 assembly를 분리해 archive에 올린다.
- BioProject/BioSample/assembly metadata를 함께 정리한다.
- 2025년 이후 NCBI submission에서는 AGP 대신 `gapped assembly` 제출 흐름을 따른다.

---

## 8. 이 문서를 실제로 쓰는 방법

다른 세션에서 `작물이 최종 확정`되면, 이 문서를 아래 순서로 바로 이어 쓰면 된다.

### 8-1. 먼저 바꿔야 하는 칸

- `최종 작물명`
- `project type (A/B/C)`
- `reference thesis statement`
- `reference line 후보`
- `핵심 형질 3개`
- `활용 use-case 3개`

### 8-2. 그다음 붙이면 되는 입력값

- 기존 assembly accession
- genome size / ploidy
- 공개 annotation 여부
- 핵심 논문 5개
- source holder/legal route 요약

### 8-3. 마지막에 꼭 확인할 것

- 이 과제가 아직도 `new reference`를 주장해야 하는가
- 아니면 `upgrade`, `local reference`, `pangenome starter`로 framing을 바꿔야 하는가

이 한 줄이 안 잠기면 이후 모든 설계가 흔들린다.

---

## 9. 현재 기준 최종 권고

현재 로컬 문서와 외부 기준을 함께 보면,  
지금 가장 필요한 것은 `새 작물 후보를 더 넓게 찾는 일`이 아니라 아래 순서다.

1. `MC1-1`을 `작물 thesis` 수준으로 닫는다.
2. 선택 작물을 `A/B/C형`으로 분류한다.
3. 그 분류에 맞는 `reference build thesis`를 먼저 고정한다.
4. 그다음에야 sequencing, assembly, annotation, breeding 활용을 설계한다.

즉, 다음 단계의 핵심은 `작물명`이 아니라 `reference strategy type`을 먼저 잠그는 것이다.
