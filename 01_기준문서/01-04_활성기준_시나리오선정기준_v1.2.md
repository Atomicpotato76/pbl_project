# 최종 시나리오 선정 기준 v1.2

## 목적

이 문서는 작물을 먼저 고정하지 않고, 먼저 지정학 시나리오를 통과시킨 뒤 그 시나리오에 맞는 작물 long list를 만들고 hard filter를 적용하기 위한 운영 기준표다.  
핵심 원칙은 다음과 같다.

- `PBL 문제상황.md` 원문을 최상위 권위로 두고, 우선순위는 `수요 증가 작물` → `reference genome 프로젝트 적합성` → `5주 기획 가능성` 순으로 해석한다.
- `이미 공개 유전체가 충분한 작물/작물군`은 우선 탈락시키고, 그 다음에만 commercial / legal / delivery를 본다.
- `commodity demand`와 `commercial seed demand`를 구분한다.
- `beneficiary`, `buyer`, `payer`, `decision-maker`를 분리한다.
- `reference-genome necessity`는 soft score가 아니라 hard gate로 본다.
- `기존 public genome 대비 novelty`를 별도 hard gate로 본다.
- 법·제도는 배경 설명이 아니라 hard gate + 공식 출처 확인 대상으로 본다.
- `대기업이면 해결 가능` 같은 heroic assumption은 baseline에 넣지 않고 sensitivity로만 보낸다.

---

## PBL-first 우선순위

이 기준표는 아래 위계로 읽는다.

1. `PBL 원문 적합성`
   최근 국내외에서 수요가 증가하는 작물인가, 그리고 그것이 `표준유전체(reference genome)` 프로젝트 기획 과제로 수렴하는가.
2. `공개 유전체 충분성 fail-fast`
   이미 공개 genome/pangenome/marker/resource로 핵심 질문이 대부분 해결되면 바로 탈락이다.
3. `5주 기획 적합성`
   실제 실험 수행이 아니라, 5주 안에 전략 기획과 30분 발표로 정리 가능한가.
4. `지원 필터`
   payer, delivery, legal/governance, assumption control은 위 1~3을 통과한 뒤 붙는 현실성 필터다.

메모:

- 위 1~3을 못 넘는 후보는 commercial attractiveness가 높아도 carry하지 않는다.
- 특히 `이미 공개 유전체가 충분하다`는 판정은 `HF-0/HF-3`에서 즉시 탈락으로 취급한다.

## A. 시나리오 선정 Hard Gate

| 코드 | 기준 | 핵심 질문 | 최소 근거 | 판정 규칙 |
|---|---|---|---|---|
| SG-1 | PBL 적합성 | 이 시나리오가 “수요 증가 작물의 reference genome 기획”으로 수렴하며 5주 기획 과제로 다룰 수 있나? | 수요 증가 근거 + reference genome project fit + 5주 기획 범위 설명 | 명백히 아니면 `N`, 일부 불명확하면 `B` |
| SG-2 | 수요 증가 및 수요 유형 명확성 | 수요가 실제로 증가하고 있나, 그리고 그 증가가 소비 증가인지, 생산 부족인지, 수입 의존인지, 정책/조달인지 구분됐나? | 공식통계 또는 공식 정책자료 | 증가 자체가 미확인이면 `UNKNOWN — official/statistical source required`, 유형 미구분이면 `B` |
| SG-3 | payer 분리 | beneficiary, buyer, payer, decision-maker가 각각 보이나? | 4주체 표기 | 하나라도 unknown이면 `B` |
| SG-4 | 상업 전환/공공 경로 | commodity demand가 seed/genomic revenue로 전환되나, 아니면 public/blended route인가? | paid path 또는 공공조달/계약 근거 | 둘 다 불명확하면 `N` |
| SG-5 | delivery 경로 명시성 | 생산·증식·등록·유통 중 실제 delivery path가 named actor 기준으로 설명 가능한가? | named delivery path + 최소 1개 fallback 또는 대체 경로 메모 | path 자체가 없으면 `N`, “파트너 구하면 됨” 수준이면 `B` |
| SG-6 | 법·제도 경로 | ABS, DSI, ITPGRFA, seed law, farmers’ rights를 공식 경로로 설명 가능한가? | 조약/정부/공식기관 링크 | 공식출처 없으면 `UNKNOWN — official source required`, 내부문서와 공식출처 충돌 시 `CONFLICT`, 명백히 막히면 `N` |
| SG-7 | genome 필요성 | 새 reference genome이 없으면 못 하거나 현저히 열등해지는 분석/육종결정이 있나? | non-substitutable genome use-case + 기존 public resource 비충분성 메모 | 기존 public genome/marker/phenotyping으로 충분하면 `N` |
| SG-8 | heroic assumption 격리 | baseline이 인수, 자회사, 대기업 내부화, 미확인 해외 네트워크 같은 가정에 기대지 않나? | assumption register | heroic이면 `B` 또는 `N` |

### 시나리오 운영 규칙

- `N` 하나라도 있으면 해당 시나리오는 폐기한다.
- `B`, `UNKNOWN`, `CONFLICT`는 보완자료 확보 전까지 long list 생성 금지다.
- 시나리오 제목에는 작물명을 넣지 않는다.
- 시나리오 카드는 최소한 아래 필드를 포함해야 한다.

```text
Scenario ID
- Region/TPE
- Demand driver
- Beneficiary
- Buyer
- Payer
- Decision-maker
- Value capture
- Delivery controller
- Legal-governance route
- Reference genome use-case
- Heroic assumptions
- Partner concentration
- Geopolitical exposure / mitigant
- Evidence status
```

## B. 작물 × 시나리오 Hard Filter

| 코드 | 기준 | 핵심 질문 | 최소 근거 | 판정 규칙 |
|---|---|---|---|---|
| HF-0 | 공개 유전체 충분성 fail-fast | 이미 공개 genome/pangenome/marker/resource로 핵심 breeding question이 사실상 해결되었나? | NCBI/Ensembl/Gramene/1차 논문 기준 현재 자원 현황 | `충분하다`고 판단되면 즉시 `N` |
| HF-1 | 시나리오 적합성 | 이 작물이 해당 시나리오에 자연스럽게 들어가나? | 1문장 fit rationale | 억지면 `N` |
| HF-2 | benchmark + must-win traits | benchmark cultivar와 must-win traits 2~4개가 정해졌나? | benchmark, trait, threshold | 없으면 `B` |
| HF-3 | 기존 자원 대비 novelty | 기존 public genome/pangenome 대비 추가 가치가 있나? | NCBI/Ensembl/Gramene/1차 논문 기준 자원 현황 + assembly/annotation/pangenome 대표성 메모 | 기존 자원으로 충분하면 `N` |
| HF-4 | genome-to-breeding bridge | reference → 분석 → breeding decision → must-win trait / value capture가 이어지나? | 4단계 causal chain | 끊기면 `N` |
| HF-5 | germplasm/legal access | germplasm access와 ABS/legal route가 공식 기준으로 설명 가능한가? | accession/법·제도 근거 | 공식출처 없으면 `UNKNOWN — official source required`, 충돌 시 `CONFLICT`, 막히면 `N` |
| HF-6 | revenue/delivery viability | hybrid/licensing/service/public route 중 최소 1개가 작동하나? | paid path + delivery owner | 없으면 `N` |
| HF-7 | genome technical feasibility | genome size, ploidy, heterozygosity, repeat content, DNA extraction, assembly/annotation workflow가 기획 범위 내에서 감당 가능한가? | public resource + literature + named workflow | 불명확하면 `B`, 과도하면 `N` |
| HF-8 | phenotyping feasibility | must-win traits를 실제로 phenotype/validation 가능한가? | protocol 또는 문헌 근거 | 약하면 `B` |
| HF-9 | geopolitical kill-switch | 전쟁, 제재, 등록불가, 물류봉쇄 같은 즉시 붕괴 리스크가 있나? | 국가/물류/정책 리스크 확인 | kill-switch면 `N` |

### 작물 × 시나리오 운영 규칙

- `B`, `UNKNOWN`, `CONFLICT`는 soft score로 보내지 않는다.
- `Y`만 shortlist 후보로 유지한다.
- `HF-0`은 최우선 fail-fast다. 공개 유전체가 이미 충분하면 그 후보는 바로 탈락이다.
- `reference-genome necessity`, `novelty`, `genome technical feasibility` 중 하나라도 `N`이면 즉시 탈락이다.
- 공공가치 시나리오는 민간 direct-sales 시나리오와 같은 점수판에서 바로 비교하지 않는다.

## C. 세일즈 포인트 공식

세일즈 포인트는 아래 순서로만 작성한다.

```text
payer의 pain point
→ benchmark 대비 must-win traits
→ genome이 unlock하는 분석
→ N사 차별화 자산
→ delivery / revenue path
→ 증거
```

### 세일즈 포인트 규칙

- `genome이 unlock하는 분석`이 약하면 세일즈 포인트가 아니라 슬로건으로 본다.
- “프리미엄”, “기능성”, “전통성”, “K-” 같은 표현은 단독으로 쓰지 않는다.
- 기존 public genome, marker, 품종, phenotyping만으로 충분하면 reference genome 세일즈 포인트로 쓰지 않는다.
- `N사 차별화 자산`은 증거가 있는 경우에만 쓴다. 원문에 없는 해외 법인, 현지 유통망, breeding station, 인수 역량은 baseline 금지다.
- 모든 세일즈 포인트에는 근거등급과 반증조건을 붙인다.

## D. 조사 흐름

1. crop-agnostic 시나리오 3~5개 생성
2. `SG-1 ~ SG-8` 적용
3. 공통 evidence-confidence 메타필드 부착
4. 통과 시나리오만 long list 생성
5. `crop × scenario` 단위로 정리
6. `HF-0`으로 공개 유전체 충분성 fail-fast prescreen
7. `HF-1 ~ HF-9` 적용
8. 공통 evidence-confidence 메타필드 재점검
9. `Y`만 shortlist로 이동
10. shortlist만 deep research 진행
11. 그 뒤 route별 soft score와 sensitivity 분석 수행

## E. 분리 원칙

- `evidence confidence`는 점수축이 아니라 모든 판단 위에 덮는 meta-layer다.
- 모든 SG/HF 판단에는 최소한 `source grade`, `critical claim`, `falsifier`, `status(Y/B/N/UNKNOWN/CONFLICT)`를 붙인다.
- `partner concentration`은 기본 점수축보다 scenario overlay에 가깝다.
- overlay는 최소한 `single partner / single country / single donor / single logistics corridor`, `substitutability`, `fallback`을 포함한다.
- `지정학 리스크`는 단일 점수보다 `kill-switch / exposure / mitigant / sensitivity` 구조로 본다.
- `대기업이면 가능`은 baseline이 아니라 sensitivity다.
- soft score는 hard gate 통과 후에만 적용한다.
- 공공/혼합 route와 민간 direct-sales route는 같은 soft-score 판에서 직접 비교하지 않는다.
