### MC2-1: 선정 작물 유전체 DEEP-DIVE 📖

- **조사 범위:** 선정 작물의 유전체 특성 심층 조사

  - assembly 난이도 판정

[배경]

[조사 항목]
1. 유전체 기본 스펙
   - genome size (Mb/Gb) — flow cytometry 또는 k-mer 기반 추정치
   - ploidy level (2n=?) — diploid / polyploid 여부
   - heterozygosity 수준 (%) — haplotype-resolved assembly 필요 여부 판단

   - GC content

2. 기존 reference genome 현황
   - 발표된 reference assembly 목록 (NCBI, Ensembl Plants, 개별 논문)
   - 각 assembly의 품질 지표: contig N50, scaffold N50, BUSCO completeness, LAI
   - 사용된 시퀀싱 기술 (Illumina only? PacBio? ONT? Hi-C?)
   - gene annotation 상태 (MAKER/BRAKER/ab initio)
   - 알려진 한계 / 공백

3. Pangenome 현황
   - graph-based pangenome 존재 여부
   - structural variation (SV) catalog 존재 여부
   - 포함된 accession 수
   - 공개 여부

4. 근연종(closely related species) 유전체 비교
   - 같은 genus 내 해독된 종 목록
   - synteny / collinearity 정보
   - wild relative 유전체 여부

5. Assembly 난이도 종합 판정
   - 부록 A Genome Feasibility Decision Tree 기준 분류
   - 예상 challenging point
   - 추천 assembly 전략 방향 (single ref vs pangenome-first)

- 기존 reference 비교표

이 특성에 맞는 최적의 시퀀싱 전략과 조립 파이프라인을 설계해야 한다.
학부 3학년 팀이 5주 기획안으로 제출하는 것이므로, 실행 가능하면서도
교수님이 "이 팀이 시퀀싱·조립을 이해하고 있다"고 판단할 수 있는 수준이어야 한다.

[Part A: 시퀀싱 전략 설계]

1. 시퀀싱 플랫폼 조합 설계
   - PacBio HiFi, ONT (ultra-long), Illumina short-read, Hi-C 중 어떤 조합?
   - 각 플랫폼의 역할 (primary assembly / polishing / scaffolding / phasing)
   - 2026년 기준 최신 chemistry (PacBio Revio, ONT Kit14/R10.4.1 등)
   - 대안 조합 1개 이상 제시 (예: HiFi+Hi-C vs ONT+Hi-C)

2. Coverage 설계
   - 각 플랫폼별 권장 coverage (×)
   - coverage가 assembly 품질에 미치는 영향
   - 비용 최적화 관점의 minimum viable coverage

3. Reference line 선정
   - 어떤 accession/cultivar를 reference로 쓸지 기준
   - inbred line vs F1 hybrid 선택 근거
   - 고분자 DNA 추출 가능성

4. 시퀀싱 비용 추정
   - 플랫폼별 대략적 비용 (2026년 기준)
   - 총 예상 시퀀싱 비용 범위

[Part B: 조립 파이프라인 설계]

5. Primary assembly
   - HiFi 기반: hifiasm (default) vs Verkko vs 기타
   - ONT 기반: Flye / Shasta / NextDenovo
   - 하이브리드: Verkko (HiFi+ONT)
   - 각 tool의 장단점 비교 (특히 이 작물의 유전체 특성에 맞는지)

6. Scaffolding
   - Hi-C scaffolding tool: YaHS / SALSA2 / ALLHiC
   - chromosome-level assembly 달성 전략

7. Polishing (필요시)
   - HiFi consensus accuracy가 충분한지
   - Illumina polishing 필요 여부

8. Quality Control
   - BUSCO completeness (기대 목표: ≥95%)
   - contig/scaffold N50 (기존 assembly 대비 개선 목표)
   - LAI (repeat annotation quality)
   - MERQURY (k-mer completeness)
   - 기타 QC metrics

9. Haplotype phasing (해당시)
   - haplotype-resolved assembly 필요 여부
   - trio binning / Hi-C phasing / 기타

[Part C: 전체 파이프라인 흐름도]

10. 위 설계를 종합하여 아래 형태의 파이프라인 흐름도 작성:
    DNA 추출 → 시퀀싱(플랫폼별) → QC → Primary assembly → Scaffolding → Polishing → QC → (Phasing) → Final assembly

[출력 형태]
- 시퀀싱 전략 비교표 (플랫폼 조합 2-3개)
- 추천 전략 1개 + 근거
- 조립 파이프라인 tool chain 표
- 전체 파이프라인 흐름도 (ASCII)
- 예상 비용 + 일정 요약
- 핵심 참고 논문/문서 5-7개

[자기 검증 — Observation]
출력 완료 후, 아래 3문항에 자체 답변하라:
(a) 추천 시퀀싱 전략(플랫폼 조합 + coverage)이 MC2-1에서 파악된 유전체 특성(genome size, repeat, het)에 맞는가? 예: repeat 60%+ 유전체에 short-read only를 추천하고 있지는 않은가?

- **조사 범위:** MC1-1 + MC2-1 + MC2-2 결과를 초안 문서로 통합

<stage_0_command>

## 블록 1: 작물 선택 + 근거 (5분)

- 선정 과정 요약 (3줄 — 상세는 부록 참조)

## 블록 2: 유전체 구조 분석 (3-5분)
- genome size, ploidy, heterozygosity, repeat content, chromosome

- 왜 새 표준유전체가 필요한지

## 블록 3: 시퀀싱 전략 (5-7분)
- 플랫폼 조합 + coverage
- reference line
- 비용 추정
- 대안 전략과의 비교

## 블록 4: 조립 파이프라인 (5분)
- tool chain (primary → scaffolding → polishing → QC)
- 파이프라인 흐름도
- 품질 목표 (BUSCO, N50, LAI)

[작성 기준]

- 각 블록 내용은 교수님 학습목표(시퀀싱 전략, 최신 방법, 조립, 유전자 예측)에 직접 연결되어야 한다
- 팀원 분업 가능 섹션 표시: [분업 가능] 태그

[출력 형태]

- 각 블록의 핵심 메시지 1줄 요약
- 부록으로 넘길 자료 목록

(b) 교수님 학습목표 6개가 블록 1-4 어딘가에 모두 반영되었는가? (학습목표 4의 "유전자 예측"은 Week 2 범위이므로 "추후 상세화" 언급 정도면 충분)
