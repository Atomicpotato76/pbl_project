# 세이프티 하네스 — 조사 전 판정 기준

문서 상태: COMMON
목적: 과확장을 막고, 대분류가 끝나기 전에 세부사항으로 빠지는 것을 방지한다.

## 1. 사용 시점

사용자가 `~에 대해 조사하자`고 말하거나, 작업 중 세부 주제가 커질 때 먼저 적용한다.

## 2. 조사 전 5문항

1. 이 조사가 실제 연구기획 또는 실험 설계 절차에 필요한가.
2. 필요하더라도 현재 블록에서 필요한가.
3. 아직 대분류가 끝나지 않았는데 세부로 파고드는 것은 아닌가.
4. 이 내용이 발표 본문에 들어갈 정보인가, 부록/Q&A 정보인가.
5. 사실관계 오류를 막기 위한 검증인가, 이미 확인된 결론을 반복하는 과검증인가.

## 3. 응답 양식

```text
이 조사는 [실무 적합/실무 부적합]입니다.
다만 현재는 [블록 n / 대분류 n-n] 단계이므로, [지금 진행/블록 완료 후 진행/부록으로 이동]이 맞습니다.
진행한다면 발표 본문에는 [핵심 결론 1개]만 남기고, 세부사항은 [부록/Q&A/삭제]로 처리하겠습니다.
```

## 4. 기본 보류 항목

- 후보 작물 전면 재탐색
- 모든 논문 수치 비교
- AI 모델별 성능 비교
- 실제 계약 수준 buyer/payer 확정
- pangenome 기술사 자체를 발표 핵심으로 확장
